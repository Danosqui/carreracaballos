Errores de Conexión a la Base de Datos

Proyecto: Carrera de Caballos — JPA + Hibernate + MySQL

🔍
Se encontraron 4 problemas que impiden que la aplicación conecte a MySQL correctamente.

Cada uno produce un error diferente. Se deben corregir en orden porque el segundo no aparece hasta que el primero está resuelto.

1

MySQL no está corriendo como servicio

Infraestructura
Error que ves
Cannot get a connection as the driver manager is not properly initialized Unable to create requested service [JdbcEnvironment]
¿Por qué pasa?
MySQL es un servidor que tiene que estar encendido antes de que cualquier programa se conecte a él. En Mac con Homebrew, no arranca automáticamente con el sistema operativo — hay que levantarlo manualmente o configurarlo para que arranque solo.
✓ Cómo solucionarlo
Abrí una terminal y ejecutá uno de estos comandos:
# Levantar MySQL una sola vez (se apaga al reiniciar la Mac) mysql.server start # O con Homebrew (recomendado, muestra el estado) brew services start mysql # Verificar que está corriendo brew services list | grep mysql # Tiene que aparecer: mysql started # Verificar que acepta conexiones mysql -u root -p -e "SELECT 1;"
Para que arranque automáticamente con la Mac
brew services start mysql # Esto registra MySQL como servicio de sistema — ya no hay que levantarlo a mano
2

Credenciales incorrectas en persistence.xml

Configuración
Error que ves
SQL Error: 1045, SQLState: 28000 Access denied for user 'root'@'localhost' (using password: YES)
¿Por qué pasa?
El archivo persistence.xml tiene password=root como valor por defecto de un ejemplo genérico. Tu instancia de MySQL tiene una contraseña diferente. MySQL rechaza la conexión porque la contraseña no coincide.
✓ Cómo solucionarlo
Editá el archivo src/main/resources/META-INF/persistence.xml y cambiá el usuario y contraseña:
Línea en persistence.xml
− <property name="jakarta.persistence.jdbc.user" value="root"/>
+ <property name="jakarta.persistence.jdbc.user" value="TU_USUARIO"/>
− <property name="jakarta.persistence.jdbc.password" value="root"/>
+ <property name="jakarta.persistence.jdbc.password" value="TU_CONTRASEÑA"/>
¿No sabés tu contraseña de MySQL?
Podés probar desde la terminal: mysql -u root -p y escribir la contraseña. Si no tenés contraseña, dejá el campo value="".
3

Nombre de Persistence Unit no coincide

Configuración
Error que ves
No Persistence provider for EntityManager named carreracaballosPU
¿Por qué pasa?
JPAUtil.java busca una unidad de persistencia llamada carreracaballosPU, pero el persistence.xml tiene definida una llamada herenciaPU (nombre copiado de un ejercicio anterior sobre herencia). Son dos strings distintos → JPA dice que no existe ningún proveedor con ese nombre.
✓ Cómo solucionarlo
Editá src/main/resources/META-INF/persistence.xml, primera línea del bloque <persistence-unit>:
persistence.xml
− <persistence-unit name="herenciaPU" transaction-type="RESOURCE_LOCAL">
+ <persistence-unit name="carreracaballosPU" transaction-type="RESOURCE_LOCAL">
Regla clave: el name del <persistence-unit> en persistence.xml tiene que ser idéntico (mayúsculas incluidas) al string que se pasa en Persistence.createEntityManagerFactory("carreracaballosPU") en JPAUtil.java.
4

Dependencia duplicada de MySQL en pom.xml

pom.xml
Warning que ves (no bloquea, pero es un problema)
WARNING: 'dependencies.dependency.(groupId:artifactId:type:classifier)' must be unique: com.mysql:mysql-connector-j:jar -> version 8.0.33 vs 9.0.0 @ line 33
¿Por qué pasa?
En el pom.xml el conector de MySQL está declarado dos veces con versiones distintas: 8.0.33 y 9.0.0. Maven no sabe cuál usar. Hoy no rompe la compilación, pero en versiones futuras de Maven sí lo hará, y puede causar comportamientos inesperados.
✓ Cómo solucionarlo
Eliminá una de las dos entradas en pom.xml. Quedate con la versión más nueva que sea compatible con tu Hibernate:
<!-- BORRAR ESTO (versión vieja) --> <dependency> <groupId>com.mysql</groupId> <artifactId>mysql-connector-j</artifactId> <version>8.0.33</version> </dependency> <!-- DEJAR SOLO ESTA --> <dependency> <groupId>com.mysql</groupId> <artifactId>mysql-connector-j</artifactId> <version>9.0.0</version> </dependency>

Lista de verificación para que funcione

Hacé estos pasos en orden antes de ejecutar el proyecto:

  • MySQL corriendo: brew services start mysql
  • Usuario y contraseña correctos en persistence.xml
  • Nombre de persistence-unit = carreracaballosPU en persistence.xml
  • Una sola dependencia de mysql-connector-j en pom.xml
